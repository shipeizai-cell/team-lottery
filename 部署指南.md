# 多人团建抽奖系统部署指南

## 🎯 系统功能概述
- ✅ **多人同时使用**：支持团队所有成员同时参与
- ✅ **每人只能抽一次**：基于用户ID/IP识别，防止重复投票
- ✅ **实时统计**：实时显示两个选项的得票情况
- ✅ **最终目标确定**：统计概率最高的选项作为团建目标
- ✅ **钉钉分享**：生成分享链接，直接发到钉钉群

## 🏗️ 生产环境部署（Supabase版）

### 需要：Supabase免费账户
Supabase提供免费的PostgreSQL数据库和API服务。

### 步骤1：创建Supabase项目
1. 访问 [supabase.com](https://supabase.com) 注册账号
2. 创建新项目（选择免费计划）
3. 获取以下信息：
   - Project URL
   - API Key (anon public key)
   - Project ID

### 步骤2：创建数据库表
在Supabase SQL编辑器中执行：

```sql
-- 创建参与者表
CREATE TABLE participants (
  id BIGSERIAL PRIMARY KEY,
  user_id TEXT UNIQUE NOT NULL,
  user_name TEXT NOT NULL,
  choice TEXT NOT NULL CHECK (choice IN ('家里蹲', '歆弘府')),
  user_ip TEXT,
  user_agent TEXT,
  created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

-- 创建索引
CREATE INDEX idx_participants_user_id ON participants(user_id);
CREATE INDEX idx_participants_choice ON participants(choice);
CREATE INDEX idx_participants_created_at ON participants(created_at DESC);

-- 启用RLS（行级安全）
ALTER TABLE participants ENABLE ROW LEVEL SECURITY;

-- 创建策略：允许插入
CREATE POLICY "允许插入投票" ON participants
  FOR INSERT WITH CHECK (true);

-- 创建策略：允许所有人读取
CREATE POLICY "允许读取投票" ON participants
  FOR SELECT USING (true);
```

  constructor() {
    this.supabaseUrl = '你的SUPABASE_URL';

    this.supabase = supabase.createClient(this.supabaseUrl, this.supabaseKey);

  async hasVoted(userId) {
    const { data, error } = await this.supabase
      .from('participants')
      .select('id')
      .eq('user_id', userId)
      .limit(1);
    
    return data && data.length > 0;
  }
  
  async submitVote(userId, userName, choice) {
    // 检查是否已投票
    if (await this.hasVoted(userId)) {
      return { success: false, message: '您已经投过票了' };
    }
    
    // 获取用户IP（通过后端函数）
    const userIp = await this.getUserIP();
    
    const { data, error } = await this.supabase
      .from('participants')

        {
          user_id: userId,
          user_name: userName,
          choice: choice,
          user_ip: userIp,
          user_agent: navigator.userAgent
        }
      ]);
    
    if (error) {
      return { success: false, message: '投票失败: ' + error.message };

## 🚨 注意事项

### 安全性
1. **API密钥保护**：不要在前端暴露敏感密钥
2. **速率限制**：防止恶意刷票
3. **数据备份**：定期备份投票数据

### 性能
1. **CDN加速**：如果用户分布广，使用CDN
2. **数据库优化**：定期清理旧数据
3. **缓存策略**：对统计数据进行缓存

### 法律合规
1. **隐私政策**：告知用户数据使用方式
2. **数据保留**：明确数据保留期限
3. **用户同意**：首次使用时请求同意

## 🆘 故障排除

### 常见问题
1. **无法投票**：检查用户ID是否唯一，检查数据库连接
2. **统计不更新**：检查Supabase API权限
3. **页面加载慢**：检查网络连接，优化图片资源
