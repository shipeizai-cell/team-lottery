# 🚀 团建抽奖系统一键安装指南

## 📦 你已经拥有的文件

在 `deploy` 文件夹中，你已经拥有：
- ✅ `index.html` - 主程序（自动配置向导）
- ✅ `supabase_setup.sql` - 数据库SQL脚本
- ✅ 所有必要的文档文件

## 🎯 30分钟完成生产部署

### 第一步：创建Supabase账户（5分钟）
1. **访问** [https://supabase.com](https://supabase.com)
2. **点击** "Start your project"
3. **注册** 使用GitHub或邮箱（推荐GitHub）
4. **创建组织** 输入公司名称
5. **创建项目**：
   - Name: `team-lottery-system`
   - Password: 设置强密码（记住它！）
   - Region: `Asia Southeast (Singapore)`
   - Plan: `Free Plan`
6. **等待** 项目创建完成（约1分钟）

### 第二步：配置数据库（5分钟）
1. **进入项目** 仪表板
2. **点击左侧** `SQL Editor`
3. **点击** `New query`
4. **复制** `supabase_setup.sql` 文件内容
5. **粘贴** 到SQL编辑器中
6. **点击** `Run` 按钮执行
7. **验证** 看到"数据库设置完成！"消息

### 第三步：获取API凭证（2分钟）
1. **点击左侧** `Settings` → `API`
2. **复制** 以下信息：
   - `Project URL` (以 https:// 开头)
   - `anon public` key (以 eyJ 开头)

### 第四步：配置投票系统（3分钟）
1. **访问** 你的 `index.html` 文件
   - 如果本地：双击打开
   - 如果服务器：访问对应URL
2. **按照向导** 一步步操作：
   - 步骤1：欢迎界面 → 点击"开始配置"
   - 步骤2：Supabase项目创建（你已完成）→ 点击"已完成，继续"
   - 步骤3：输入API凭证 → 粘贴URL和Key → 点击"测试连接"
   - 步骤4：自动配置数据库 → 点击"配置数据库"
   - 步骤5：配置完成 → 点击"启动投票系统"

### 第五步：部署到服务器（5分钟）
选择一种部署方式：

#### 方案A：上传到公司服务器
1. 将整个 `deploy` 文件夹上传到Web服务器
2. 确保可通过URL访问（如 `http://公司服务器/抽奖/`）

#### 方案B：使用GitHub Pages（免费）
1. 创建GitHub仓库
2. 上传 `deploy` 文件夹内容到仓库
3. 启用GitHub Pages
4. 访问 `https://用户名.github.io/仓库名`

#### 方案C：云存储（推荐）
- 腾讯云COS / 阿里云OSS
- 开启静态网站托管
- 上传文件，获取访问URL

### 第六步：钉钉分享（2分钟）
1. **复制** 你的投票系统URL
2. **打开** 钉钉群
3. **发送** 以下消息：

```
【团建投票开始】🏢

投票决定团建地点：
🏠 家里蹲 - 温馨舒适
🍽️ 歆弘府 - 豪华餐厅

规则：每人一票，实时统计
投票：你的URL链接

截止时间：今天 20:00
```

## 🆘 故障排除

### 问题1：Supabase连接失败
✅ 解决：
- 检查URL格式：必须 `https://xxx.supabase.co`
- 检查Key是否正确复制（完整无空格）
- 确认网络可以访问Supabase

### 问题2：数据库SQL执行错误
✅ 解决：
- 检查是否已有同名表（删除重试）
- 确认有创建表的权限
- 逐段执行SQL，找出错误位置

### 问题3：投票页面无法访问
✅ 解决：
- 检查文件路径是否正确
- 确认服务器运行正常
- 尝试用不同浏览器访问

### 问题4：投票不保存
✅ 解决：
- 检查Supabase连接状态
- 查看浏览器控制台错误
- 确认数据库表权限设置正确

## 📊 系统管理

### 查看投票数据
1. 登录Supabase
2. 进入 `Table Editor`
3. 查看 `participants` 表
4. 可以导出CSV格式数据

### 重置投票
如果需要重新投票：
```sql
-- 在Supabase SQL编辑器中执行
TRUNCATE TABLE participants RESTART IDENTITY;
```

### 监控使用情况
- Supabase免费版限制：500MB数据库，2GB带宽
- 一般团队投票完全够用
- 可在Supabase仪表板查看使用量
